# PI Planning

Domain knowledge for SAFe Program Increment planning preparation.

## When to Use

- User asks to prepare for PI Planning or a planning ceremony
- Breaking features into PI-sized increments
- Prioritizing features using WSJF
- Mapping cross-team dependencies

## PI Planning Components

- Feature breakdown (fits within one PI, 8-12 weeks)
- WSJF scoring (Cost of Delay / Job Size)
- Dependency mapping (cross-team, cross-ART)
- PI Objectives (SMART goals per team)
- Risk identification (ROAM: Resolved, Owned, Accepted, Mitigated)

## Key Principles

- Features must fit within one PI. If not, split further.
- WSJF prioritizes by economic value, not opinion or politics.
- Dependencies are risks — minimize them, don't just document them.
- Load SAFe terminology from `references/domain-knowledge.md`.
- PI Objectives are commitments — distinguish committed vs uncommitted.
- Enablers (technical work) compete for capacity alongside business features.
